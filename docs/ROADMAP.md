# 開發藍圖 (Roadmap)

本文件紀錄了 Maker IDE 接下來的開發階段與具體功能目標，主要分為「完善基礎功能」、「優化使用者體驗」與「專案延伸」三個階段。

---

## 🎯 階段 1：完善 Arduino IDE 基礎功能 (核心基礎)

此階段著重於將現有的編輯器打造成具備完整官方 IDE 基礎能力的開發環境。

1. **圖形化函式庫管理員 (Library Manager)**
   * 提供友善的介面搜尋、瀏覽並一鍵安裝/移除 Arduino 函式庫。
   * 支援版本選擇與依賴自動處理。
2. **圖形化開發板管理員 (Board Manager)**
   * 列出官方與第三方開發板（如 ESP32, ESP8266）。
   * 支援在背景自動下載並安裝對應的核心編譯套件。
3. **內建範例程式庫 (Examples)**
   * 內建或從後端載入官方與第三方函式庫的經典範例程式碼 (如 Blink, WebServer 等)。
   * 讓使用者能一鍵載入範例進行測試。
4. **進階序列埠終端與除錯 (Terminal Serial Debug)**
   * 強化目前的 Serial Monitor。
   * 新增 Serial Plotter（序列埠繪圖家）功能。
   * 支援設定不同的鮑率 (Baud rate)、顯示時間戳記，以及自動捲動控制。

---

## 🌟 階段 2：優化使用者體驗 (UI/UX 升級)

在基礎功能完備後，進一步提供符合現代化軟體水準的操作體驗。

1. **頂部選單列與手動存檔 (Menu Bar & Manual Save)**
   * 實作跨平台統一的自繪頂部選單列 (File, Edit, View 等)。
   * 加入防呆全域快捷鍵 (`Ctrl+S` / `Cmd+S`)。
   * **手動存檔策略 (理想操作)**：
     * **純網頁版 (Web)**：存檔動作即為觸發瀏覽器下載 `.ino` 原始碼檔案。
     * **桌面端版 (Electron)**：存檔動作為呼叫系統對話框選擇路徑，自動建立同名資料夾並在內部寫入 `.ino` 檔案，符合官方 IDE 規範。
2. **本地自動存檔與專案草稿管理 (Auto-save & Workspace)**
   * **自動存檔**：編輯時於背景自動儲存目前的程式碼，避免網頁重整或意外關閉後遺失。
   * **多專案管理**：支援儲存多個不同的「專案草稿 (Sketches)」並提供側邊欄切換介面。
   * **雙環境存儲策略**：
     * **純網頁版 (Web)**：使用 `localStorage` 或 `IndexedDB`。
     * **桌面端版 (Electron)**：整合 Node.js `fs`，直接寫入作業系統實體資料夾 (`AppData` 或自訂路徑)。
3. **支援多檔案專案 (Multi-file)**
   * 在編輯介面加入檔案樹與頁籤 (Tabs)。
   * 支援建立、切換與共同編譯 `.ino`、`.cpp`、`.h` 檔案。
4. **多國語系與自訂主題 (i18n & Customize Theme)**
   * **i18n**：導入多國語系架構，支援繁體中文、英文等語言切換。
   * **自訂主題**：支援深色 (Dark)、淺色 (Light) 模式，允許自訂 Monaco Editor 的程式碼高亮顏色配置。

---

## 🚀 階段 3：專案延伸 (擴展生態系)

結合更多的教育與開源社群資源，讓工具發揮更大價值。

1. **整合積木寫程式 (Blockly)**
   * 引入 Google Blockly，提供視覺化積木程式介面。
   * 實現「拖曳積木即時產生 C++ 程式碼」的雙向或單向轉換，非常適合程式初學者與 STEM 教育。
2. **功能介紹 Markdown (Documentation)**
   * 在專案中（或透過靜態網頁）撰寫完整的使用教學、功能特色介紹 Markdown。
   * 作為專案的官方文件或 Landing Page 內容，幫助新使用者快速上手。
