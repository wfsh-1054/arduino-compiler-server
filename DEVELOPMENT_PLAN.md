# Arduino 雲端 IDE 開發計劃

這份文件記錄了 Arduino Compiler Server 專案的未來發展方向與待辦事項。目前專案已經具備「編輯 -> 雲端編譯 -> 網頁無縫燒錄」的 MVP（最小可行性產品）核心功能，接下來的目標是提升使用者體驗與擴展更多硬體支援。

## 🎯 階段 1：編輯器與介面優化（UI/UX 升級）

目前的介面僅有基礎的 `<textarea>`，我們需要將其升級為具備現代 IDE 體驗的介面。

- [ ] **導入 Monaco Editor**：
  - 替換現有的 `<textarea>`，使用 Monaco Editor (VS Code 的核心編輯器)。
  - 實作 C/C++ 語法高亮 (Syntax Highlighting)。
  - 增加基礎的自動完成 (Auto-completion) 與括號匹配。
- [ ] **改善 UI 佈局**：
  - 建立左右分割畫面：左側為程式碼編輯區，右側為操作面板與終端機輸出日誌。
  - 美化按鈕狀態與視覺回饋（例如加入載入動畫 Spinner）。
- [ ] **加入 Serial Monitor (序列埠監控視窗)**：
  - 利用現已連接的 Web Serial API 資源，建立一個可讀取並顯示 Arduino `Serial.print()` 輸出的監控視窗。
  - 實作向下捲動與清除輸出功能。

## 🎯 階段 2：後端與編譯效能優化

隨著功能增加，後端的穩定性與效能需要進一步加強。

- [ ] **編譯快取機制 (Caching)**：
  - 目前每次編譯都會重新編譯整個核心庫。我們應在 `arduino-cli` 編譯指令中加入 `--build-cache-path`，讓相同的板子核心檔能被重複利用，大幅縮短第二次之後的編譯時間。
- [ ] **加入 ESP32 及其他開發板支援**：
  - 雖然目前 API 已預留 `esp32` 參數，但前端還未提供選項，且 Dockerfile 尚未安裝 ESP32 核心。
  - 在前端加入下拉選單，讓使用者動態選擇 `Arduino Uno` 或 `ESP32` 等板子。
  - 擴展 Dockerfile 與 `arduino-cli` 的核心庫安裝腳本。
- [ ] **編譯錯誤訊息格式化**：
  - 目前 `arduino-cli` 回傳的錯誤訊息可能較為雜亂，可在後端解析並過濾出關鍵的錯誤行數與原因，再回傳給前端。

## 🎯 階段 3：進階功能與服務化

- [ ] **函式庫管理員 (Library Manager)**：
  - 允許使用者在網頁上搜尋並勾選需要安裝的 Arduino 函式庫 (透過 `arduino-cli lib install`)。
- [ ] **程式碼存檔與雲端同步**：
  - 結合 LocalStorage，讓使用者的程式碼在重新整理頁面後不會消失。
  - 遠期目標：加入簡單的帳號系統或 GitHub Gist 整合，讓程式碼可存放於雲端。
- [ ] **多檔案專案支援**：
  - 支援除了單一 `.ino` 以外的專案架構（例如加入自定義的 `.cpp` 或 `.h` 標頭檔）。

## 📝 備註

- 目前專案已移除 mDNS 廣播機制，聚焦於單機 / 雲端容器化部署與 Web Serial API 燒錄的穩定性。
